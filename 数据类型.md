## 清空表数据的方式

```mysql
1 delete from test4;
2 truncate table test4;
```

```mysql
区别在使用 truncate table在功能的使用上和delete相同，但是truncate只能清空所有表的数据，但是delete后可以使用where条件判断进行删除指定的数据。

truncate操作比delete要快，delete会产生空间碎片，而truncate不会。因为truncate删除的同时会释放表空间，而delete不会释放，truncate占用的系统和事务日志资源较少。

当表数据被清空时，使用truncate清空的表，索引和自增值都将恢复成初始值的大小，而delete不会。
```

```mysql
1.  ‘0’或者‘00’ - ‘69’ 被识别为2000-2069

```

## time数据类型写入规则

```mysql
字符串或者数值按照三组数据写入（时分秒），正常识别
当有两组数据时，被识别为时分
当有一组数据时，被识别为秒
当带有空格时，空格前面的数值被识别为天数，当空格后面有三组数据时，正常识别时分秒，
有两组数据时，被识别为时分，只有一组数据时，识别为小时。
当0写入时，被识别为00:00:00
```
